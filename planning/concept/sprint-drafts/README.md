# Successor Two-Week Sprint Drafts

The Concept Studio is responsible for keeping the next two-week development sprint drafted, clarified, and ready for promotion before the current sprint ends.

## Responsibility

While the current sprint is executing, the Concept Studio continuously incorporates:

- Accepted concept decisions;
- current-sprint implementation evidence;
- validation and performance boundaries;
- observed player-comprehension findings;
- unresolved risks and dependencies; and
- the current Day 10 Continue, Narrow, or Stop decision structure.

The result is a complete successor sprint proposal, not an automatically active roadmap.

## Readiness Timeline

- **By Day 2:** create or identify the successor draft and state its product question.
- **By the end of Day 5:** incorporate the first weekly concept findings and list unresolved blocking choices.
- **By Day 7:** draft work items, dependencies, estimates, acceptance criteria, validation, risks, and cuts.
- **By Day 8:** reach Candidate Complete.
- **On Day 9:** reconcile with the user and current development evidence.
- **Before Day 10 closes:** mark Ready for Promotion or record an explicit blocker and fallback plan.

## Promotion Boundary

A successor draft must be marked **Draft/Conditional** while it lives here. It becomes active only after:

1. the current sprint records its closing decision;
2. the user approves the successor direction;
3. current repository truth and validation gates are reconciled; and
4. the plan is deliberately promoted into `planning/active/`.

If a Draft/Conditional successor already exists under `planning/active/`, do not create a competing duplicate. Treat that file as the existing successor candidate and use weekly handoffs to recommend changes until the main development track authorizes edits.

Use [the two-week sprint template](../templates/TWO-WEEK-SPRINT-TEMPLATE.md) for new successors.
